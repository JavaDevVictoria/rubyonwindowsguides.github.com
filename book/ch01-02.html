Improving Command Prompt

In the previous chapter we’ve learned how to enable Unicode characters in the Command Prompt, how to change and create environment variables with the emphasis on the system path. Before we explore all “secrets” of installing Ruby on Windows we will make few more improvements in the Command Prompt.

Every new Command Prompt on Windows is opened as new window. It is quite impractical if you need to open several Command Prompts. Linux and OSX users have benefit because in one terminal window they can open shells as multiple tabs which make work in the shell much more comfortable. Wouldn’t it be good to have such a feature on Windows too?

There are few commercial applications which we will omit here and will focus on the Open Source solution. Console application) is the one we will use. Currently it is in the 2.0 beta stage but it is quite stable for everyday use. As a matter of act I use it and I never faced any crash or malfunction. You can choose 32 or 64 bit version depending on the version of Windows you have. At the time of writing this book version is 2.00.147. Console2, as this version is more frequently referred to, comes as a Zip archive. Just unpack it anywhere on the disk and start console.exe. New window with standard Command Prompt will open. You can use menu to open new Command Prompt in a new tab.

Console2 has a lot of options which help you to customize it to your needs. Besides background and foreground colors, fonts and other visual customizations you can define hot keys for opening new tabs, switch between them, activating specific tab and many more which we will not discuss here. But one option deserves more attention. Tabs section in Settings dialog is used to define which shell will be opened in new tabs. By default Console2 opens tab with cmd.exe shell, but you can use others like Cygwin or MSYS bash shells. This will come in handy later when we install Git and DevKit.

Command Prompt on Windows is capable of displaying colored text. Unfortunately Microsoft implementation differs from standard so ANSI color codes, which are used in all Ruby extensions, are not interpreted correctly. For a long time win32console extension was used on Windows platforms for converting ANSI color codes in the form Command Prompt understands. Although it did great job at the time and most of extensions were able to use win32console, it had one drawback – slowness.

At the moment of writing this book recommendation is to use ANSICON or ANSICON GitHub Project, application that hooks into cmd.exe process on Windows, parses ANSI color codes and converts them so Command Prompt can displayed colored output. Like Console2 ANSICON comes as Zip archive. In order to install it you have to unpack it to some folder on your path. There are few ways ANSICON can be used. If ansicon.exe is just started new command processor (typically cmd.exe) is started. With option -p ANSICON hooks in the current Command Prompt and recognition of ANSI escape sequences is enabled. If you want ANSICON to be enabled whenever you start new Command Prompt you should invoke it with:

ansicon -i or
ansicon -I

The first one adds ANSICON to the AutoRun registry value for current user and the second for local machine.
